注文を表す集約ルートエンティティ

com.example.demo.domain.model.order.Order

実店舗におけるテーブル単位の注文を管理する。 注文は複数の注文明細（{@link OrderItem}）を持ち、 ステータスに応じた状態遷移のビジネスルールを保持する。

集約の不変条件

  • 注文には1つ以上の注文明細が必要
  • ステータス遷移は {@link OrderStatus} の遷移規則に従う
  • キャンセル済み・提供済みの注文は変更不可
フィールド フィールド型

status 注文ステータスを表す列挙型
orderedAt LocalDateTime
メソッド 引数 戻り値型 説明
+ 注文をキャンセルする void
+ 注文ステータスを更新する 注文ステータスを表す列挙型 void

ステータスの遷移は {@link OrderStatus#canTransitionTo(OrderStatus)} の ルールに従う。不正な遷移を行った場合は例外がスローされる。
+ 注文IDを取得する 注文IDを表す値オブジェクト
+ 注文明細リスト（不変）を取得する List<注文明細を表す値オブジェクト>
+ 注文日時を取得する LocalDateTime
+ 注文ステータスを取得する 注文ステータスを表す列挙型
+ テーブル番号を取得する テーブル番号を表す値オブジェクト
+ この注文がアクティブ（未完了）かどうかを判定する boolean
+ 合計金額（税抜）を計算する 金額を表す値オブジェクト
+ 合計金額（税込）を計算する 金額を表す値オブジェクト
staticメソッド 引数 戻り値型 説明
