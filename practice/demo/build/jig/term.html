common 共通基盤パッケージ

本パッケージは、ドメインに依存しない横断的な共通機能を提供する。 全レイヤーで利用すべき基盤機能がここに集約されている。

提供する共通機能一覧

クラス役割利用必須レイヤー
{@link com.example.demo.common.logging.AppLogger} アプリケーションログ出力 全レイヤー（webapi / application / query / infrastructure）
{@link com.example.demo.common.exception.BusinessException} 業務例外の基底クラス domain / application
{@link com.example.demo.common.exception.SystemException} システム例外の基底クラス infrastructure
{@link com.example.demo.common.date.DateUtils} 日付・時刻操作の汎用関数群 全レイヤー

新規参画者向け

本プロジェクトでは、ログ出力に {@code System.out.println} や 素の {@code LoggerFactory.getLogger} を直接使用せず、 必ず {@link com.example.demo.common.logging.AppLogger} を使用すること。 これは ArchUnit テストで強制されている。

共通ユーティリティの検索方法

全ての共通ユーティリティには {@link com.example.demo.common.SharedUtility @SharedUtility} アノテーションが付与されている。 IDEで「{@code @SharedUtility}」を検索すれば、利用可能な共通機能を一覧できる。

date 日付操作ユーティリティパッケージ

本パッケージは、プロジェクト全体で統一的に使用する日付・時刻操作機能を提供する。 Java標準の {@link java.time} APIをラップし、業務で頻出する日付操作を簡素化する。

提供クラス

  • {@link com.example.demo.common.date.DateUtils} - 日付操作の汎用関数群

使用ルール

  • 日付操作は必ずこのパッケージのユーティリティを使用すること
  • {@code new Date()} や {@code SimpleDateFormat} は使用禁止
  • タイムゾーンは {@code Asia/Tokyo} が標準

logging アプリケーションログ出力パッケージ

プロジェクト全体で統一されたログ出力を提供する。 直接 SLF4J の {@code Logger} を取得せず、 {@link com.example.demo.common.logging.AppLogger} を使用すること。

使用例

{@code
public class OrderApplicationService {

    public OrderId placeOrder(PlaceOrderCommand command) {
        log.info("注文を受け付けました: テーブル{}", command.tableNumber());
        // ...
        log.warn("在庫残りわずか: {}", menuItemId);
    }
}
}

なぜ共通ロガーを使うのか

  • ログフォーマットの統一（トレースID・操作コンテキストの自動付与）
  • ログレベルポリシーの一元管理
  • 将来的な監視基盤との連携を容易にする

DateUtils 日付操作ユーティリティ

プロジェクト全体で統一的に使用する日付・時刻の汎用関数を提供する。 すべてのメソッドは {@link java.time} APIベースで、スレッドセーフである。

使用方法

{@code
// 今日の日付を取得
LocalDate today = DateUtils.today();

// 現在日時を取得
LocalDateTime now = DateUtils.now();

// 日付フォーマット
String formatted = DateUtils.formatDate(today);         // "2026/02/28"
String withTime  = DateUtils.formatDateTime(now);       // "2026/02/28 14:30:00"

// 営業日判定
boolean open = DateUtils.isBusinessDay(today);          // 平日ならtrue

// 日付の差分
long days = DateUtils.daysBetween(startDate, endDate);  // 2つの日付間の日数

// 期間判定
boolean within = DateUtils.isWithinDays(targetDate, 7); // 7日以内か判定
}

設計方針

  • すべて static メソッドとして提供（インスタンス化不要）
  • タイムゾーンは {@code Asia/Tokyo} を標準とする
  • null引数に対しては {@link IllegalArgumentException} をスローする
  • 旧来の {@code java.util.Date} や {@code SimpleDateFormat} は使用しない
 クラス com.example.demo.common.date.DateUtils

AppLogger アプリケーション共通ロガー

プロジェクト全体でログ出力を統一するためのラッパークラス。 全てのログ出力はこのクラスを経由して行うこと。

使用方法

{@code
// クラスフィールドとして宣言
private static final AppLogger log = AppLogger.of(MyClass.class);

// ログ出力
log.info("処理を開始します: {}", param);
log.warn("想定外の状態: {}", state);
log.error("処理に失敗しました", exception);
}

ログレベルポリシー

  • {@code DEBUG} - 開発時のデバッグ情報（本番では出力しない）
  • {@code INFO} - 業務処理の正常な実行記録（注文受付、ステータス変更等）
  • {@code WARN} - 想定内だが注意が必要な事象（在庫不足、リトライ等）
  • {@code ERROR} - システムエラー・予期しない例外
